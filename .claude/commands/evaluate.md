# 評価AIスキル - ページ品質評価

作成したWebページを専門的な観点から評価・採点します。

## 使用方法

```
/evaluate [ファイルパスまたはURL]
```

引数がない場合は、現在のプロジェクトの主要ページを評価します。

## 評価基準

### 1. SEO評価 (25点満点)

- メタタグ（title, description, keywords）の最適化
- 見出し構造（H1〜H6）の適切さ
- 画像alt属性の設定
- 構造化データ（JSON-LD）の実装
- URL構造・パーマリンクの最適化
- ページ読み込み速度への配慮
- モバイルフレンドリー対応

### 2. 事業内容の明確さ (25点満点)

- サービス内容が一目で分かるか
- 何ができるサイトなのか明確か
- ターゲットユーザーが明確か
- 競合との差別化ポイントが分かるか
- 信頼性を示す要素があるか

### 3. 分かりやすさ (25点満点)

- 文章が簡潔で読みやすいか
- 専門用語に適切な説明があるか
- ビジュアル（図解・スクリーンショット）の活用
- ナビゲーションが直感的か
- 情報の階層構造が適切か

### 4. サービス申込意欲 (25点満点)

- CTAボタンの配置・デザイン
- 利用のメリットが明確か
- 利用手順が分かりやすいか
- 不安要素の解消（FAQ等）
- 今すぐ使いたくなる訴求力

```

2. 事業内容の明確さ:   XX/25点
3. 分かりやすさ:       XX/25点
4. サービス申込意欲:   XX/25点

【良い点】
- ...
- ...

【改善が必要な点】
- ...
- ...

【具体的な改善提案】
1. ...
2. ...
3. ...

【優先度の高い修正】
□ [修正1]
□ [修正2]
□ [修正3]
```

## 合格基準

- **合格ライン**: 70点以上
- **優秀ライン**: 85点以上
- **不合格**: 70点未満（再度調整が必要）

## 実行例

```bash
# 特定ファイルを評価
/evaluate index.html

# URLを評価
/evaluate http://localhost:3000

# プロジェクト全体を評価
/evaluate
```

## 自動修正モード

評価後、改善提案に基づいて自動修正を行うことも可能です：

```
/evaluate --fix index.html
```

## 他のスキルとの連携

- **/persona** - ユーザー視点での評価と組み合わせる
- **/codex** - 改善コードの自動生成
- **/gemini** - SEOコンテンツの最適化
